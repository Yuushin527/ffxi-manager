### Windower ランチャー問題の解決

**経緯：**
- mouse_event でクリック → 無効（フック or DPI 問題）
- PostMessage WM_LBUTTONDBLCLK → 無効
- SendInput Enter → 未確認だが無効

**解決策：** `settings.xml` の `profile name=""` を `name="default"` に変更し、`Windower.exe -p default` でランチャー UI をスキップ

→ POL が直接起動するようになった ✅

### 現在の状態と残課題

| 項目 | 状態 |
|------|------|
| Windower ランチャースキップ | ✅ 解決（-p default）|
| chars bin 自動切り替え | ✅ 動作確認済み |
| フラグファイル書き出し | ✅ 動作確認済み |
| POL 自動ログイン | ❌ chars1.bin の autologin フラグが 0（無効） |
| FFXI ToS・キャラ選択 | 未テスト（POLで止まるため） |

**POL 自動ログインが無効の原因：** chars1.bin offset 0x6F = 0

**対策として決定した方針：**
- キャラ1人につき1つの bin ファイルを用意する
- 各 bin ファイルは POL で「記憶する」ログインしてキャプチャ
- 6キャラ分（chars1_akris.bin 等）を順番にセットアップする
- autoPOL config.json で各キャラの charsBin を個別に指定
