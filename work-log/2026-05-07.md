### Windower ランチャー自動クリック（→ボタン）

- `-p "Default Profile"` 引数でランチャーをスキップしようとしたが、プロフィール読み込みが機能しないことが判明
  - 空白なしプロフィール名（`Default`）も試したが同様
  - `-p` フラグで指定した場合、Windower が別プロフィールを生成し addons が読み込まれない
- 最終方針：`-p` フラグは使わず、ランチャーの → ボタンをマウスクリックで自動操作
  - `ShowWindow + SetForegroundWindow + ClientToScreen` でクライアント座標→スクリーン座標変換してクリック
  - `settings.xml` の profile name は `""` （元の状態）に戻した
- **【重大バグ】** `simulateKey(VK_RETURN)` でランチャー操作を試みた際、グローバルキー送信が CMD ウィンドウに届いてしまい autoPOL が無限起動するバグを発生させた → `simulateKey` 方式を完全排除
- 現在の方式：`GetClientRect → ClientToScreen → mouse_event` で座標クリック（91% x, 95% y）。**テスト未実施**

### ToS・メニュー・キャラ選択の自動化

- デバッグログ追加で根本原因を特定：
  - POL ウィンドウは約 13 秒で別ウィンドウへ遷移（タイトルが空になる）
  - `Final Fantasy XI` ウィンドウは**別の hwnd** として新規作成される
  - → 既存の `hwnd` を使い回す方式では検出不可
- `FindWindowW(NULL, L"Final Fantasy XI")` で新ウィンドウを検出する方式に変更
- キーボード（SendInput / PostMessage / keybd_event）はすべて無効：
  - Windower の `WH_KEYBOARD_LL` フックが注入キー（`LLKHF_INJECTED`）を透過しない
  - `keyboard_allowmmf 1`（init.txt に記載）が有効な環境ではキーボードが通る可能性あるが未確認
- **マウスクリック方式に切り替え**（`ClientToScreen` + `mouse_event`）
  - ToS「同意する」: client (787, 476)
  - メインメニュー「キャラクター選択」: client (672, 519)
  - キャラリスト「Akris」（1番目）: client (755, 332)
  - `BlockInput(FALSE)` 解除後に FFXI ウィンドウ検出 → クリック実行
- **テスト未実施**（就寝のため翌日へ持ち越し）
