# FFXI Manager — Claude Code 作業ルール

## プロジェクト概要

Windows WPF アプリ（C# / .NET 9）。FFXI の複数アカウント・プロフィール管理ツール。

## セッション管理コマンド

### 始めて（セッション開始）

以下を順番に実行する：

1. `local-config.md` を読んで使用中の PC を確認する
2. `todo.md` を読んで現在のタスク状況を表示する
3. 今日の作業ログファイル `work-log/YYYY-MM-DD.md` を作成する（存在する場合はスキップ）
4. `git status` と直近の `git log` を確認して現在の状態を把握する
5. 上記をまとめてユーザーに報告する

### 締めて（セッション終了）

以下を順番に実行する：

1. 今日の `work-log/YYYY-MM-DD.md` に作業内容を記録する
2. `todo.md` を最新状態に更新する（完了タスクを移動、新規タスクを追加）
3. コミットしていない変更があれば確認してユーザーに提示する
4. 作業サマリーをユーザーに報告する

## TODO コマンド

`todo.md` は以下の3セクションで管理する：

- **未着手** — まだ始めていないタスク
- **進行中** — 作業中のタスク
- **完了** — 完了したタスク

「TODO: ○○を追加して」と言われたら未着手に追加。「○○が完了した」と言われたら完了に移動。

## 確認が必要な操作

以下は必ずユーザーに確認してから実行する：

- `git push`（リモートへの反映）
- ファイルの削除
- 破壊的な操作全般

## 作業ログのフォーマット

```
PC: {local-config.md の PC 名}

## YYYY-MM-DD 作業内容

### {作業タイトル}

- {実施内容}
- {実施内容}
```

## 技術スタック

- 言語: C# 13 / .NET 9
- UI: WPF (XAML)
- アーキテクチャ: MVVM
- ビルド: `dotnet build`
- テスト: `dotnet test`
